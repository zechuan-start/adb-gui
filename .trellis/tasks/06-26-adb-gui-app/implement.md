# 实施计划: ADB GUI MVP 重规划

## 当前规划修正

- [x] 将产品重新定位为桌面 Android 开发测试工具, 不是控制台式项目.
- [x] 将完整 app 列表从 P0 核心降级为 P1/P2 可选辅助能力.
- [x] 从 MVP 要求中移除左侧侧边栏信息架构.
- [x] 确认截图行为: 保存到本机 + 外部查看器打开, 不做内置预览.
- [x] 将适合 GUI 化的 adb 能力按 P0/P1/P2/危险区分层.

## 阶段 1: 将现有代码对齐到新 MVP 形态

- [ ] 将当前 `App.tsx` 的侧边导航替换为应用外壳:
  - 顶部 `DeviceContextBar`.
  - 默认 `QuickToolsWorkspace`.
  - 只有保留 P1 工具入口时才使用顶部 segmented control.
- [ ] 将截图移动到默认工作台.
- [ ] 将 APK 安装保留在默认工作台.
- [ ] 将快捷按键工具加入默认工作台.
- [ ] 将当前应用动作工具加入默认工作台.
- [ ] 将 Logcat 和 Apps 移到非默认 tab, 或从 P0 构建中移除.
- [ ] 确保默认 UI 不再出现左侧侧边栏.

验证:

- [ ] 启动 app, 确认首屏是工具工作台, 不是侧边栏控制台.
- [ ] 确认截图, APK 安装不用切换模块即可看到.
- [ ] 确认快捷按键和当前应用动作不用完整 app 列表即可使用.

## 阶段 2: 设备上下文

- [ ] 保持后端 adb 路径解析为唯一事实源.
- [ ] 在顶部上下文栏展示 adb 来源和版本.
- [ ] 设备选择器支持刷新, 并清楚展示连接状态:
  - 无设备.
  - 在线.
  - 未授权.
  - 离线.
  - 多设备.
- [ ] 当前 Activity 在顶部上下文栏展示, 并支持手动刷新.

验证:

- [ ] 无设备连接时, P0 操作禁用且状态清楚.
- [ ] 单个在线设备连接时, 操作指向该设备.
- [ ] 多设备连接时, 操作前能明确看到选中设备.

## 阶段 3: 截图工具

- [ ] 后端将 `adb exec-out screencap -p` 输出保存到系统图片目录的 `ADB GUI` 文件夹.
- [ ] 文件名包含安全设备标识和时间戳.
- [ ] 后端通过系统默认查看器打开已保存 PNG.
- [ ] 前端展示保存路径.
- [ ] 前端提供复制路径和在文件管理器中显示.
- [ ] 从 MVP 中移除任何内置预览要求或预览组件.

验证:

- [ ] 截图生成有效 PNG.
- [ ] PNG 会外部打开.
- [ ] macOS 上路径复制和显示文件可用.
- [ ] Windows 打开/显示文件行为有设计覆盖或手工验证清单.

## 阶段 4: APK 安装工具

- [ ] 可选择 APK 文件并安装到选中设备.
- [ ] 可拖拽 APK 到工作台并安装.
- [ ] 展示 stdout/stderr 安装结果.
- [ ] 无在线选中设备时禁用安装.

验证:

- [ ] 有效 APK 可安装.
- [ ] 无效 APK 失败时展示 adb 错误.
- [ ] 拖拽安装路径可用.

## 阶段 5: 快捷按键工具

- [ ] 定义按键允许列表:
  - 返回.
  - 主页.
  - 最近任务.
  - 回车.
  - 电源.
  - 音量加/减.
- [ ] 实现 `adb shell input keyevent` 后端包装.
- [ ] 在默认工作台增加紧凑图标按钮.
- [ ] 无在线选中设备时禁用按钮.

验证:

- [ ] 返回/主页/最近任务在真机可用.
- [ ] 前端不能发送未允许的 key code.

## 阶段 6: 当前应用动作工具

- [ ] 从当前 Activity 解析当前包名.
- [ ] 实现当前包强停.
- [ ] 实现清数据/缓存, 并要求确认.
- [ ] 实现当前包启动.
- [ ] 实现当前包卸载, 并要求确认.
- [ ] 不依赖完整 app 列表.

验证:

- [ ] Activity 切换时当前包会更新.
- [ ] 强停能停止当前包.
- [ ] 清数据/卸载需要确认, 并展示 adb 输出.

## 阶段 7: P1 能力收口

- [ ] 如果现在保留 Logcat, 放到顶部 tab `日志`.
- [ ] 如果现在保留 AppManager, 放到顶部 tab `应用`, 且不是默认页.
- [ ] App list 不在启动时主动拉取, 除非用户打开可选 tab.
- [ ] App list 不是截图, 文本传递或 APK 安装的前置条件.

验证:

- [ ] 启动不被包列表拉取阻塞.
- [ ] 默认工作台仍聚焦 P0 工具.

## 阶段 8: 验证

- [ ] `pnpm lint` 或项目等价前端检查.
- [ ] `pnpm build`.
- [ ] `pnpm tauri build` 或记录平台特定替代命令.
- [ ] macOS + 真机最小冒烟:
  - 选择设备.
  - 查看当前 Activity.
  - 截图并外部打开.
  - 安装 APK.
  - 点击返回/主页/最近任务.
  - 强停当前包.
- [ ] 截图保存重新设计后的桌面 UI, 供复核.

## 风险点

- Tauri 文件打开/显示 API 存在跨平台差异, 先明确行为并优先验证 macOS.
- App list/logcat 代码可能已存在, 但现有实现不能反向绑架 MVP 布局.
- 清数据和卸载是破坏性动作, 必须确认并展示目标包名.
- 当前应用启动方式会因包而异, 无法解析 launcher activity 时必须显示明确错误.

## 继续实现前需要确认

- [ ] 用户确认 Logcat 是留在 P0 还是移动到 P1.
- [ ] 用户确认快捷按键和当前应用动作是否进入 P0.
- [ ] 用户复核新版 PRD/design 后批准继续实现.
