# AI Agent 双设备自动化测试

## 目标

构建独立的双设备协同 App 自动化测试框架. 用 YAML 定义测试流程骨架, AI Vision Agent (Midscene.js) 在两台 Android 设备上分别扮演 "执行者" 和 "质量核查者", 乒乓式串行完成测试执行与结果验证.

## 用户价值

- 测试用例用自然语言描述, 降低自动化测试编写门槛.
- 双设备协同覆盖需要两个用户角色配合的业务场景 (执行 + 质量核查).
- YAML 骨架保证流程可控可复现, AI 自主操作适应界面变化.
- CLI 运行, 可集成 CI/CD.

## 已确认事实

- 独立 Node.js 项目, 不集成进 adb-gui.
- 两台设备跑同一个 App, 登录不同账号 (执行角色 + 质量角色).
- 协同模式: 乒乓式串行.
  - 执行端按任务单表单完成当前步骤 → 质量端核查并完成 → 执行端刷新继续下一步 → 循环.
- 测试用例格式: YAML 定义流程骨架 (步骤顺序, 角色, 自然语言目标), AI Agent 自主规划具体 UI 操作.
- 核心框架: Midscene.js (视觉驱动, JS 生态, 支持多模型, AndroidWorld 93.1%).
- 视觉模型: 云端 API 起步 (Doubao-Seed / Qwen / Gemini 任选), 不需要本地 GPU.

## 需求

### MVP - 最小可运行版本

- 测试用例定义:
  - YAML 文件描述测试流程, 包含: 名称, 设备配置, 前置条件, 有序步骤列表.
  - 每个步骤包含: 角色 (executor/quality), 自然语言操作描述, 可选断言.
  - 支持步骤间等待条件 (视觉等待: 等某个元素出现).

- 双设备 Agent 管理:
  - 通过 ADB serial 绑定两台设备, 分别创建 Midscene AndroidAgent 实例.
  - 启动时验证两台设备均在线且可连接.

- 乒乓编排引擎:
  - 按 YAML 步骤顺序执行, 根据 role 字段分发到对应设备 Agent.
  - 每步执行: aiAct(action) → 可选 aiAssert(assert) → 截图存档.
  - 步骤间支持 waitFor 视觉等待 (等另一台设备操作的结果同步).
  - 任一步骤失败: 截图 + 标记失败 + 可配置继续或终止.

- 测试报告:
  - 每步记录: 步骤描述, 角色, 设备, 通过/失败, 截图路径, 耗时.
  - 运行结束生成 JSON 结果文件.
  - 复用 Midscene 内置可视化报告 (每步操作回放).

- CLI 入口:
  - `npx ai-test run cases/create-task.yaml` 执行单个用例.
  - `npx ai-test run cases/` 批量执行目录下所有用例.
  - 支持 `--executor-serial` / `--quality-serial` 命令行指定设备.

### P1 - 增强

- 测试用例模板和变量替换 (同一流程不同数据).
- 从测试管理系统拉取用例.
- 失败重试 (单步或整个流程).
- 并行执行多个独立测试用例 (多组设备).
- Web UI 查看历史报告.

## 验收标准

- [ ] 可通过 YAML 定义包含 executor 和 quality 两个角色的测试流程.
- [ ] 两台 Android 设备各绑定一个 Midscene Agent, 按步骤乒乓执行.
- [ ] 每步执行后自动截图, 可选断言验证.
- [ ] 步骤间支持视觉等待 (等 UI 元素出现).
- [ ] 任一步骤失败时截图并标记, 生成失败报告.
- [ ] CLI 可运行单个或批量 YAML 用例.
- [ ] 生成包含每步截图和通过/失败状态的测试报告.

## 不做范围

- 不集成进 adb-gui, 独立项目.
- 不做 Web UI (MVP 阶段).
- 不做本地模型部署, 云端 API 起步.
- 不做 iOS / HarmonyOS 支持 (MVP 只做 Android).
- 不做录屏, 只做逐步截图.
